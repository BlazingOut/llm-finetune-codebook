## **一、 本章目标**

在动手写微调代码之前，我们需要先解决一个极其现实的物理问题：**显存不足 (OOM, Out of Memory)**。 完成本章理论学习后，你将理解：

1. 为什么全量微调一个大模型会撑爆你的消费级显卡？
2. LoRA 是如何通过“矩阵分解”的数学魔法，将训练参数量减少 99% 的？
3. QLoRA 中的“4-bit 量化”是如何把显存占用压缩到极致的？


## 二、 显存账单：为什么不能直接“全量微调”？

在深度学习的早期，如果我们想让一个模型学习新知识，通常的做法是**全量微调 (Full Fine-Tuning)**——即把模型里所有的参数都解冻，让它们在训练中全部发生改变。

但对于大语言模型（LLM）来说，这是一笔极其昂贵的“显存账单”。我们以 **Qwen2.5-1.5B** 为例来算一笔账：

|**组件**|**物理含义**|**显存占用 (近似值)**|
|---|---|---|
|**模型权重 (Weights)**|模型本身的参数（以 16-bit 精度加载）|~ 3 GB|
|**梯度 (Gradients)**|训练时记录每个参数该往哪个方向修改的数值|~ 3 GB|
|**优化器状态 (Optimizer)**|比如 Adam 优化器，需要记录过去的梯度状态（动量和方差，占用 32-bit）|~ 12 GB|
|**激活值 (Activations)**|前向传播时产生的中间结果（与输入文本长度成正比）|~ 2-4 GB|
|**总计显存需求**|**一次标准的全量微调**|**> 20 GB**|

> **📉 结论：** 即便是一个只有 15 亿参数的“小”模型，全量微调也需要至少 **24GB** 的显存（如 RTX 3090/4090）。对于只有 **8GB 或 12GB** 显存的普通显卡（如 RTX 3060）来说，直接运行会立刻报错：`CUDA Out of Memory`。

## 三、 LoRA 的物理直觉：“不拆大厦，只搞软装”

为了在消费级显卡上训练大模型，微软的研究人员提出了一种名为 **LoRA (Low-Rank Adaptation, 低秩微调)** 的天才算法。

## 1. 核心思想

如果把预训练好的大模型比作一栋**已经建好的摩天大楼**（包含丰富的通用知识）：

- **全量微调**：相当于把大楼的承重墙全砸了重新砌。成本极高，且容易把楼搞塌（即模型遗忘掉原本的通用能力，这叫“灾难性遗忘”）。
    
- **LoRA 微调**：大楼的承重墙（原始权重）**全部冻结，绝对不改**。我们只在大楼的外墙上搭几个极轻便的“脚手架”（附加的旁路矩阵）。模型在推理时，同时走承重墙和脚手架。
    

## 2. LoRA 的数学魔法

在代码层面，大模型里的每一层（比如上一章我们看到的 Self-Attention 层）本质上都是一个巨大的参数矩阵 $W$。

LoRA 的做法是保持 $W$ 不变，在它旁边并联两个很小的矩阵 $A$ 和 $B$。新的计算公式变为：

$$W_{new} = W + \Delta W = W + A \times B$$

**这为什么能省参数？**

假设原始矩阵 $W$ 的维度是 $10000 \times 10000$：

- **全量微调**：你需要训练 $10000 \times 10000 = 100,000,000$（1 亿）个参数。
    
- **LoRA 微调**：我们设定一个极小的**秩 (Rank)**，通常记为 $r$（比如 $r=8$）。
    
    - 矩阵 $A$ 的维度降为 $10000 \times 8$
    - 矩阵 $B$ 的维度降为 $8 \times 10000$        
    - 你只需要训练 $(10000 \times 8) + (8 \times 10000) = 160,000$（16 万）个参数。

**参数量直接减少了 99.8%！** 这意味着梯度和优化器状态占用的显存也随之暴跌。

## 1. 实现 LoRA：只需 5 行代码

在代码中，实现 LoRA 的核心思路是：**先加载一个普通模型，然后用一个“包装盒”把它套起来。** 这个包装盒就是 `peft`（Parameter-Efficient Fine-Tuning）库。

```python
from peft import LoraConfig, get_peft_model

# 1. 准备 LoRA 的配置说明书
lora_config = LoraConfig(
    r=8,                                # 秩 (Rank)：决定脚手架的厚度
    lora_alpha=16,                      # 缩放系数：控制微调权重的影响力
    target_modules=["q_proj", "v_proj"],# 目标模块：要把脚手架搭在哪里？
    lora_dropout=0.05,                  # 随机丢弃率，防止死记硬背（过拟合）
    bias="none",                        # 通常不训练偏置项
    task_type="CAUSAL_LM"               # 任务类型：因果语言模型
)

# 2. 见证奇迹的时刻：把普通模型包装成 LoRA 模型
peft_model = get_peft_model(original_model, lora_config)

# 打印一下看看我们省了多少参数
peft_model.print_trainable_parameters()
```

#### 🔍 核心函数与参数深度解码：

- **`LoraConfig`**：用来配置构造lora块需要的参数。
    - **`r` (Rank)**：这是最重要的参数。`r` 越大，模型能记住的新细节越多，但显存占用和训练时间也会增加。对于简单任务（如教它拍马屁），`r=8` 足矣；对于复杂的角色扮演，通常设为 `16` 或 `32`。
    - **`target_modules`**：在 Qwen 等 Transformer 模型中，注意力机制（Attention）包含了 `q_proj`、`k_proj`、`v_proj` 和 `o_proj`。我们通常优先将 LoRA 挂载在计算量最大的 `q_proj` 和 `v_proj` 上。

- **`get_peft_model`**：这是一个“物理改造”函数。它在底层做了两件事：
    
    1. 冻结 `original_model` 的所有原有参数（将 `requires_grad` 设为 `False`）。
    2. 按照配置，实例化 $A$ 和 $B$ 两个小矩阵，并将它们拼接到原模型上，设定为可训练（`True`）。
