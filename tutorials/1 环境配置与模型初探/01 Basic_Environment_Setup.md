#### 1.1 为什么选择它？

- **环境隔离**：不同的 AI 项目（如 Llama 3 和 Qwen 2.5）可能依赖不同版本的软件包。如果全部安装在一起，会发生严重的“库冲突”。
- **轻量高效**：Miniconda 是 Anaconda 的精简版，去掉了大多数初学者用不到的预装软件，只保留了最核心的虚拟环境管理功能，安装极快且占用空间小。

#### 1.2 安装指南

由于不同操作系统的安装细节有所不同，请参考以下详尽教程进行安装：

> **注意**：安装完成后，你就拥有了一个可以运行 Python 代码的“后台”。具体的指令操作，请参考本章 **【三、走进命令行世界】**。

#### 1.3 虚拟环境实操“三部曲”

安装完成后，打开你的终端（Windows 用户请搜索并打开 **Anaconda Prompt**），依次输入以下命令：

Bash

```
# 1. 创建一个名为 "llm-finetune" 的隔离环境，并指定 Python 版本为 3.10 
conda create -n llm-finetune python=3.10 -y 

# 2. 激活（进入）这个新创建的环境

```

# 三、走进命令行世界

对于初学者来说，“终端”和“命令行”就像是计算机的“后台入口”。虽然我们习惯了用鼠标点点点（图形界面），但在深度学习和服务器操作中，键盘输入才是最高效的沟通方式。

---

## 1. 终端、命令行与 Shell：如何与计算机“对话”

在微调模型的过程中，你会频繁听到这几个词。它们虽然经常被混用，但其实分工明确：

### 1.1 什么是终端 (Terminal)

**终端是“翻译官”的窗口。**

它是一个程序，为你提供了一个输入文字和显示文字的界面。
* **在 Windows 上**：常用的终端有 `PowerShell`、`命令提示符 (CMD)` 或者你安装 Miniconda 后出现的 `Anaconda Prompt`。
* **在 Linux/Mac 上**：就叫 `Terminal`。
* **比喻**：终端就像是手机的**短信对话框**，它是你和计算机发消息的那个界面。
### 1.2 什么是命令行 (Command Line)

**命令行是“对话的内容”。**

它指的是你在终端里输入的那一串特定的文字指令（例如 `cd`、`ls`、`python train.py`）。
* **特点**：计算机通过识别这些预定义的单词来执行任务，而不是靠你点击图标。

### 1.3 什么是 Shell？

**Shell 是背后的“逻辑大脑”。**

当你从终端输入一行命令行后，Shell 负责解释这行文字的意思，并告诉操作系统去干活。
* 常见的 Shell 有：`bash` (Linux 默认)、`zsh` (Mac 默认) 等。

---

## 2. 为什么微调模型一定要用命令行？

很多初学者会问：为什么不能做个按钮让我点，非要背这些命令？

1.  **远程控制**：服务器（如 AutoDL、阿里云）通常没有显示器，也没有 Windows 那样的桌面。你只能通过 SSH 连接后，用文字指令来操控它。
2.  **效率与自动化**：你可以写一个脚本（`.sh` 文件），里面包含 100 条命令，让它半夜自动执行。如果用鼠标点，你得在电脑前点 100 次。
3.  **精确控制**：命令行可以带很多“参数”。比如下载模型时，你可以通过参数告诉它“只下载权重文件”或“断点续传”，这在图形界面里很难做得这么细致。

## 4. 认识 Conda 专用终端
Anaconda 自带的终端（通常称为 **Anaconda Prompt**​ 或 **Anaconda Terminal**）是一个预配置好的命令行工具，它核心的功能是**自动激活 base 环境**，让你无需手动操作就能直接使用 `conda`和 `pip`等命令来管理Python环境和包。
这里就是我们接下来的教程中**输入指令的地方**。

**与系统默认终端的区别**

| 对比项           | Anaconda 终端 (Anaconda Prompt)   | 系统默认终端 (CMD / PowerShell / Bash) |
| ------------- | ------------------------------- | -------------------------------- |
| **启动后状态**​    | 自动位于 `(base)`环境                 | 位于系统默认环境                         |
| **conda 命令**​ | 直接可用                            | 通常需要手动配置 PATH 或运行激活脚本后才可用        |
| **主要用途**​     | **专为 Conda 环境管理设计**，进行包安装、环境创建等 | 通用系统操作，如需管理 Conda 需额外步骤          |
### 4.1 Windows 用户

在 Windows 中，标准的 `CMD` 或 `PowerShell` 默认是不认识 `conda` 命令的（除非你手动配置了复杂的环境变量）。
安装完 Miniconda 后，你的电脑上会多出几个蓝色的图标。
![[conda_prompt_.png]]
- **正确操作**：点击 Windows “开始”菜单，搜索并打开 **"Anaconda Prompt (miniconda3)"**。
- **特征**：打开后，你会发现每行文字的最左边有一个括号，写着 `(base)`。这代表你已经成功进入了 Conda 的管理世界。

### 4.2 Linux / Mac 用户：直接使用系统 Terminal
![[how_to_find_linux_terminal.png]]

# 四、Python 环境管理策略

在 Python 开发中，“环境”是一个核心概念。对于初学者来说，理清 **Base 环境** 与 **虚拟环境** 的关系，是避免后续代码运行报错的关键。

---
## 1. 什么是 Python 环境？

简单来说，一个 Python 环境就是一个**独立的文件夹**，其中包含：

1. **Python 解释器**：即 `python.exe` 执行文件。
2. **标准库**：Python 自带的功能模块。
3. **第三方库**：你通过 `pip` 或 `conda` 安装的包（如 `transformers`, `torch` 等）。

**为什么要区分环境？**

不同的项目对库的版本要求不同。例如：项目 A 需要 `torch 1.12`，而项目 B 需要 `torch 2.4`。如果只有一个环境，安装新版本就会覆盖旧版本，导致其中一个项目无法运行。

---
## 2. 什么是 Base 环境？

当你安装好 Miniconda 或 Anaconda 后，系统会自动创建一个默认的环境，这就是 **Base 环境**。

- **地位**：它是 Conda 的“根环境”或“主环境”。
- **作用**：它主要用于运行 Conda 程序本身的管理命令。
- **不建议的操作**：**强烈建议不要在 Base 环境中安装任何开发用的第三方库。**
    - 如果 Base 环境因为库冲突而损坏，可能会导致整个 Conda 无法正常工作，甚至需要重装。
    - Base 环境通常包含很多系统级插件，版本更新非常保守，不适合多变的深度学习项目。

---
## 3. 什么是用户创建的虚拟环境？

虚拟环境（Virtual Environment）是从 Base 环境中“隔离”出来的独立空间。

- **独立性**：你在虚拟环境中安装、删除或更新任何库，都不会影响 Base 环境，也不会影响其他的虚拟环境。
- **可复制性**：你可以轻松地删除一个搞乱了的虚拟环境并重新创建一个，这比重装整个 Python 要快得多。
- **纯净性**：每个环境只安装该项目必需的库，保持环境轻量且高效。

---
## 4. 常见的操作逻辑

在实际开发流程中，我们遵循以下规范：

1. **打开终端**：默认进入 `(base)` 环境。
![[conda_base.png]]

2. **创建新环境**：针对新项目（如微调大模型）创建一个专属环境。

```bash
	#  conda create -n <新环境的名字> python=指定的版本
    conda create -n llm-finetune python=3.10
```
    
3. **激活环境**：从 `(base)` 切换到 `(llm-finetune)`。

```bash
    conda activate llm-finetune
```
  ![[terminal_activate_env.png]]
4. **安装依赖**：在当前激活的环境下安装 `torch` 等工具。此时这些库只存在于 `llm-finetune` 文件夹下。

```bash
pip install <具体库的名称>
// 或者
conda install <具体库的名称>
```


# 五、可能发生的报错

## 1. 创建环境失败

> Connection broken: IncompleteRead(7923862 bytes read, 1382997 more expected)

此问题通常由网络波动或默认 conda 频道（服务器在国外）连接速度慢引起。可以尝试以下方法解决：
1. **直接重试**：有时网络问题只是暂时的。直接在终端中**重新运行一次完全相同的创建命令**即可。
2. **配置国内镜像源（推荐）**：为 conda 配置国内的镜像服务器（如清华、中科大镜像），可以极大提升下载速度和稳定性。配置后，再进行环境创建。
- 以配置清华镜像为例，执行以下命令：
```bash
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --set show_channel_urls yes
```
